# Agent 生命周期语义

这份文档专门澄清几个很容易混在一起的概念。

核心原则是：

- “禁用”是管理动作
- “断开”是运行时控制动作
- “离线”是状态结果
- “恢复”是重新接入
- “删除记录”是资产清理动作

## 1. 禁用 Agent

含义：

- 后端把持久化 Agent 资产标记为禁止接入
- 这是管理策略状态，不是删除动作
- 禁用后不能继续注册，也不能接收新任务

接口：

- `POST /agents/:agent_id/disable`
- `POST /agents/:agent_id/enable`

## 2. 断开 Agent

含义：

- 后端请求当前在线会话退出控制平面
- 这是运行时动作，不是删除动作

接口：

- `POST /agents/:agent_id/disconnect`

## 3. Agent 离线

含义：

- Agent 不再处于在线会话状态
- 这是一种状态结果，不是管理动作
- 可能由断开、崩溃、网络中断、进程退出导致

## 4. Agent 恢复

含义：

- 同一个逻辑 Agent 再次接入后端
- 它会重建新的在线会话
- 但仍然对应同一个 `agent_id`

## 5. 删除 Agent 记录

含义：

- 删除持久化 Agent 资产记录
- 这是资产清理动作，不是运行时断开

接口：

- `DELETE /agents/:agent_id`

推荐约束：

- 只在 Agent 已离线时执行

## 6. 为什么这些概念必须分开

如果把这些语义混成一坨，前后端就会很快出问题：

- 页面不知道该显示“禁用”还是“离线”
- 后端无法明确某次操作到底是清理记录还是切断连接
- 任务失败和会话消失会被误判为删除

## 7. 前后端对齐建议

前端展示时，至少保留三种层面：

- 连接状态
- 管理状态
- 资产存在性
