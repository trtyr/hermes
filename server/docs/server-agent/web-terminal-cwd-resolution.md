# 命令会话中的 CWD 解析规范

这份文档定义无 PTY 命令会话中，`cwd` 应如何解析和规范化。

## 1. 问题背景

如果前端提交：

```bash
cd ../
```

而后端只是把字符串直接拼到当前 `cwd` 上，就会出现错误路径。

## 2. 正确原则

`cwd` 的解析必须由后端或 Agent 侧执行，而不是前端自己做路径运算。

原因：

1. 前端只是 UI，不知道目标主机的路径规则
2. 前端无法访问目标主机文件系统
3. 会话状态的权威来源在后端和 Agent

## 3. 解析规则

### 3.1 绝对路径

例如：

```bash
cd /etc/app
```

应直接替换当前 `cwd`。

### 3.2 相对路径

例如：

```bash
cd ../folder
```

应基于当前 `cwd` 计算新路径。

### 3.3 规范化

必须正确处理：

- `..`
- `.`
- 多余斜杠
- 尾部冗余分隔符

### 3.4 可选真实路径解析

如果目标系统支持，建议进一步使用：

- `realpath`
- `std::fs::canonicalize`

来解析符号链接后的真实路径。

## 4. 验收示例

若会话起始目录为：

```text
/Users/alice/projects/test
```

则：

- `cd ..` -> `/Users/alice/projects`
- `cd ../../` -> `/Users/alice`
- `cd ./foo` -> `/Users/alice/projects/test/foo`

## 5. 事件流要求

后续推送给前端的命令结果或会话更新事件中，`cwd` 应使用规范化后的结果。
