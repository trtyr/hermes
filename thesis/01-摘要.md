# 摘要

随着网络安全攻防对抗走向常态化，命令与控制（C2）系统在红蓝演练中扮演的角色越来越关键，但现有工具的问题也很突出——商业框架价格昂贵且破解版存在供应链隐患，开源工具的流量特征已被安全厂商深度分析并提取指纹，传统C/C++编写的Agent则长期受困于内存泄漏和崩溃风险。本文针对上述问题，设计并实现了一套基于Rust语言的Windows平台C2框架。系统采用三层分布式架构：服务端基于Tokio异步运行时构建了微内核控制平面，以消息总线驱动状态管理与任务调度，承担Agent会话管理、监听器动态启停、交互式终端会话及WebSocket实时事件推送等核心职能；Web控制端采用Vue.js框架开发，实现了多标签终端、文件操作、截屏采集和监听器配置等操作界面；Agent端运行于Windows平台，借助Rust的所有权机制从编译阶段消除内存安全隐患，完成了TLS加密通信、抖动心跳、Shell命令执行、文件传输与代理转发等功能的开发。整条通信链路采用TLS加密，结合心跳抖动算法降低流量模式被识别的风险。持久化层选用SQLite，对Agent档案、任务记录和审计日志进行了结构化管理。经测试，系统在高并发场景下运行稳定，Agent资源占用低，达到了预期设计目标。

**关键词：** C2框架；Rust语言；Windows平台；微内核架构；网络安全

# Abstract

As offensive and defensive cybersecurity operations become increasingly routine, Command and Control (C2) systems play an increasingly critical role in red-blue exercises. However, existing tools suffer from prominent issues: commercial frameworks are costly and cracked versions carry supply chain risks, open-source tools have had their traffic signatures deeply analyzed and fingerprinted by security vendors, and agents written in C/C++ remain plagued by memory leaks and crashes. This thesis addresses these problems by designing and implementing a Windows-platform C2 framework based on the Rust programming language. The system adopts a three-tier distributed architecture: the server builds a microkernel-based control plane on top of the Tokio async runtime, using a message bus to drive state management and task dispatch, handling core functions such as agent session management, dynamic listener control, interactive terminal sessions, and real-time WebSocket event push; the web client is developed with Vue.js, providing a multi-tab terminal, file operations, screenshot capture, and listener configuration interfaces; the agent runs on Windows, leveraging Rust's ownership model to eliminate memory safety issues at compile time, and implements TLS-encrypted communication, jittered heartbeat, shell command execution, file transfer, and proxy forwarding. The entire communication path is protected by TLS encryption, combined with a heartbeat jitter algorithm to reduce the risk of traffic pattern detection. SQLite is used as the persistence layer for structured management of agent records, task logs, and audit trails. Testing confirms that the system runs stably under high-concurrency scenarios with low agent resource usage, meeting the design objectives.

**Keywords:** C2 Framework; Rust; Windows Platform; Microkernel Architecture; Cybersecurity
