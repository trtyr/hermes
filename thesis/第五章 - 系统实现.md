5 系统展示

本章以实际操作流程为主线，展示Hermes平台各功能模块的运行效果。本章从部署环境开始，依次展示Web控制台各功能页面、Agent端运行状态以及通信流量抓包结果。

5.15 通信流量验证

为验证服务端与Agent之间JSON协议的正确性，使用Wireshark在服务端网络接口抓取通信流量包[18]，逐一分析关键协议交互。

【图5-43：Wireshark - Agent注册报文】
Fig.5-43 Wireshark — Agent Registration Packet

展示Agent首次连接时发送的Register JSON报文，包含主机名、用户名、操作系统、架构、内网IP、权限级别和心跳间隔等字段。服务端回复Ack JSON消息确认注册成功，报文包含分配的session ID。

【图5-44：Wireshark - 心跳报文】
Fig.5-44 Wireshark — Heartbeat Packet

展示Agent按配置间隔周期性发送的Heartbeat JSON报文。服务端随即回复Ack确认，两个报文之间的时间差即心跳往返时间。可观察到实际心跳间隔接近配置值并存在随机抖动，防止服务端过载。

【图5-45：Wireshark - 任务分发与结果回传】
Fig.5-45 Wireshark — Task Dispatch and Result

展示完整的任务执行流程：服务端下发ExecuteTask JSON报文（含任务ID和命令字符串），Agent执行完毕后回传TaskResult JSON报文（含任务ID、执行成功标志、stdout/stderr和退出码）。两个报文可通过任务ID字段一一对应。

【图5-46：Wireshark - 终端会话数据流】
Fig.5-46 Wireshark — Terminal Session Data Flow

展示交互式终端会话中的全双工通信：服务端下发ExecuteCommandSession报文（含命令字符串），Agent回传CommandSessionResult报文（含stdout、stderr、退出码和更新后的工作目录）。多个命令的往返报文序列清晰展示了终端会话的请求-响应流式交互特征。
